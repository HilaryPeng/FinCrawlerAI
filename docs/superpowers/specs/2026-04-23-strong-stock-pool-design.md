# 强势股池双通道设计（2026-04-23）

## 背景

当前市场日报的个股识别逻辑，仍然带有较重的 `dragon / center / follow` 角色驱动痕迹。这个结构的问题不是局部排序，而是入口就偏了：

- 它先试图定义角色，再决定谁能进池。
- 它容易把真正强的票和解释性标签混在一起。
- 它要求过早判断“题材核心”“容量核心”“情绪核心”，导致边界越来越复杂。

本轮讨论后，目标被重新收敛为：

- 先找出真正值得观察的强势股。
- 再解释这些票是因为什么强。
- 第一版先解决“识别强票”，暂不把“题材归因”做得过重。

## 目标

- 建立一个统一的强势股池，先回答“这只票够不够强”。
- 入池逻辑兼容趋势强票和情绪强票，不遗漏单一风格的强票。
- 分类标签只负责解释，不反向决定入池资格。
- 主列表不再按板块配额硬凑，而是按统一强度排序。
- 页面补充板块聚集度摘要，用于观察强势股是否集中在同一方向。

## 非目标

- 本次不恢复旧的 `dragon / center / follow` 主框架。
- 本次不把 `theme_core` 作为第一版主输出。
- 本次不把板块强弱作为个股入池前置门槛。
- 本次不直接落地为代码实现，当前只沉淀设计。

## 方案比较

### 方案 A：继续沿用角色优先框架

做法：

- 先判断 `theme_core / capacity_core / emotion_stock` 等角色。
- 再按角色规则决定是否入池。

问题：

- 角色边界复杂，容易陷入规则膨胀。
- 解释标签和准入逻辑耦合过深。
- 不利于先拿到“真正强票”的最小结果。

### 方案 B：只做单一趋势池

做法：

- 所有入池都按 `5日 / 10日 / 20日` 趋势评分。
- 不单独给情绪票留入口。

优点：

- 规则最简单。

问题：

- 会漏掉纯情绪主导但仍值得观察的强票。
- 对当前市场并非总是成立，风格变化时鲁棒性不够。

### 方案 C：统一强势股池 + 双通道入池 + 解释性标签

做法：

- 先建立统一强势股池。
- 入池采用 `趋势通道 + 情绪通道` 双通道。
- 进池后再挂 `趋势强 / 情绪强 / 容量强` 标签解释。

优点：

- 先保证识别强票，再解释风格。
- 兼容趋势票和情绪票。
- 结构清晰，第一版边界最稳。

## 推荐方案

采用方案 C：`统一强势股池 + 双通道入池 + 解释性标签`。

原因：

- 它把“找强票”和“解释强票”拆开了。
- 它能覆盖趋势强票和情绪强票两类主要来源。
- 它避免第一版就把 `theme_core`、`capacity_core` 做成复杂准入条件。

## 设计

### 1. 总体结构

第一版个股层输出分成两层：

1. `强势股池`
2. `解释性标签`

这里的原则是：

- 先判断这只票是否足够强，是否值得进池。
- 再判断这只票更像趋势强、情绪强还是容量强。

标签是解释层，不是准入层。

### 2. 强势股池入池规则

强势股池采用双通道：

- `趋势通道`
- `情绪通道`

一只股票命中任一通道即可入池；同时命中两个通道时，只入池一次，但可带多个解释标签。

### 3. 趋势通道

趋势通道第一版口径如下：

- `当日成交额 >= 15亿`
- 计算 `5日 / 10日 / 20日` 趋势强度
- 至少两个周期达到中等强度
- 剩余周期不能明显转弱
- 只看个股自身，不要求板块前排

收益周期按最近 N 个可用交易日的收盘价计算，含当日。若任一周期缺少足够交易日数据，第一版不命中趋势通道，可由情绪通道补充进入强势股池。

趋势分档如下：

```text
5日涨幅 <= 0%          -> 0分
0% < 5日涨幅 <= 5%     -> 30分
5% < 5日涨幅 <= 10%    -> 60分
10% < 5日涨幅 <= 15%   -> 85分
5日涨幅 > 15%          -> 100分

10日涨幅 <= 0%         -> 0分
0% < 10日涨幅 <= 8%    -> 30分
8% < 10日涨幅 <= 15%   -> 60分
15% < 10日涨幅 <= 20%  -> 85分
10日涨幅 > 20%         -> 100分

20日涨幅 <= 0%         -> 0分
0% < 20日涨幅 <= 12%   -> 30分
12% < 20日涨幅 <= 25%  -> 60分
25% < 20日涨幅 <= 35%  -> 85分
20日涨幅 > 35%         -> 100分
```

趋势周期权重确定为：

```text
5日  = 20%
10日 = 35%
20日 = 45%
```

趋势综合分：

```text
trend_score = 0.20 * trend_5d_score
            + 0.35 * trend_10d_score
            + 0.45 * trend_20d_score
```

趋势通道命中条件：

```text
当日成交额 >= 15亿
AND trend_score >= 65
AND 5日 / 10日 / 20日中至少两个周期分数 >= 60
AND 剩余周期分数 >= 30
```

设计意图：

- 保留短中期强势信息。
- 更强调 `10日 / 20日`，避免把短脉冲票抬得过高。
- 不把板块约束作为第一版前置门槛，先把个股识别做好。

### 4. 情绪通道

情绪通道第一版口径如下：

- `当日成交额 >= 10亿`
- 满足以下任一条件即可命中：
  - `当日涨停`
  - `连板 >= 2`

情绪分第一版只看涨停与连板高度：

```text
未涨停且连板 < 2 -> 不命中情绪通道
当日涨停           -> 70分
连板 = 2           -> 85分
连板 >= 3          -> 100分
```

若同一只票同时满足当日涨停和连板条件，取更高的情绪分。

设计意图：

- 避免纯情绪强票被趋势规则漏掉。
- 先采用最小可解释规则，不在第一版加入过多情绪细分项。
- 情绪通道门槛低于趋势通道，以保留一部分典型情绪票。

### 5. 统一强度分

趋势通道和情绪通道必须合并到同一个排序分，避免两个通道各自排序后无法比较。

第一版统一强度分定义为：

```text
base_score = max(trend_score, emotion_score)

capacity_bonus =
  当日成交额 >= 100亿 -> 10分
  当日成交额 >= 50亿  -> 8分
  当日成交额 >= 20亿  -> 5分
  其他               -> 0分

multi_channel_bonus =
  同时命中趋势通道和情绪通道 -> 5分
  其他                     -> 0分

strong_score = min(base_score + capacity_bonus + multi_channel_bonus, 100)
```

排序规则：

```text
strong_score DESC
amount DESC
trend_score DESC
emotion_score DESC
symbol ASC
```

设计意图：

- 先尊重最强的入池理由，而不是把趋势和情绪机械相加。
- 成交额只做小幅加成和次级排序，避免大成交额弱票压过真正强票。
- 同时命中趋势和情绪时给小幅奖励，体现强度共振。

### 6. 主列表与数量控制

主列表不再按板块配额凑池，而是：

- 统一按强度排序
- 动态入池
- 设置安全上限 `30只`

动态入池规则：

```text
1. 股票命中趋势通道或情绪通道，即进入候选集。
2. 候选集数量 <= 30 时，全部保留。
3. 候选集数量 > 30 时，按 strong_score 排序，只保留前 30 只。
```

这里的重点是：

- 不再机械补齐非核心票。
- 允许当天强票多时多留一些，强票少时少留一些。
- 通过上限控制页面可读性和后续分析成本。

### 7. 解释性标签

第一版展示层只保留 3 个解释标签：

- `趋势强`
- `情绪强`
- `容量强`

规则如下：

- 一只股票可以同时带多个标签。
- 标签不互斥。
- 标签不决定是否入池。

标签的作用仅是帮助阅读结果：

- `趋势强`：说明这只票主要依靠中短期趋势进入强势股池。
- `情绪强`：说明这只票主要依靠涨停、连板等情绪特征进入强势股池。
- `容量强`：说明这只票具备较强的大资金承接能力。

### 8. 容量强的定位

`容量强` 第一版只做解释标签，不单独决定入池。

当前已确认：

- 它主要看绝对成交额，第一版命中条件为 `当日成交额 >= 20亿`。
- 板块内成交额地位只做辅助说明，不作为标签硬条件。
- 前面讨论过的 `capacity_core` 精细规则，先保留为第二阶段候选方案。

也就是说，第一版不再强推：

- `capacity_core` 主标签
- 板块内断层截断
- 更复杂的容量核心名额规则

而是先保留一个更直接的解释标签：`容量强`。

### 9. 板块聚集度摘要

第一版不把板块强弱作为个股入池门槛，但需要在结果层展示强势股是否集中。

强势股池生成后，按 `primary_board_name` 聚合：

```text
board_strong_count        = 该板块入池股票数量
board_strong_amount       = 该板块入池股票成交额合计
board_avg_strong_score    = 该板块入池股票 strong_score 均值
board_top_strong_stock    = 该板块 strong_score 最高股票
```

展示层可以按以下规则标记板块聚集度：

```text
board_strong_count >= 3
AND board_strong_amount >= 50亿
```

命中时，只说明“强势股在该板块集中”，不直接等同于 `theme_core` 或正式主线确认。

### 10. 第一版不单独输出 `theme_core`

当前设计明确：

- 第一版先不单独输出 `theme_core`

原因：

- `theme_core` 很容易把“强势识别”和“题材归因”混在一起。
- 题材归因对新闻稳定性、板块口径和市场语义依赖更高。
- 当前第一优先级是把强势股识别链路做稳。

后续如果要恢复 `theme_core`，应作为第二阶段增强项处理。

## 数据与排序含义

第一版排序应体现以下原则：

- 趋势通道优先识别持续性强票。
- 情绪通道补充趋势逻辑可能漏掉的强票。
- 容量是解释维度，不在第一版单独形成独立入池通道。
- 板块信息在第一版作为聚集度摘要和辅助理解信息，不作为硬门槛。
- 第一版能回答“哪些股票强”和“强票是否集中”，但不直接回答“谁是题材核心”。

## 第二阶段候选增强

本轮讨论后，以下内容明确不进入第一版，但可以作为后续增强方向：

- 恢复或重建 `theme_core`
- 将 `capacity_core` 从解释标签升级为正式角色
- 引入板块强度对个股入池的联动约束
- 引入更细的情绪强度评分，而不仅是 `涨停 / 连板`
- 引入按市场环境自适应的强势股池截断规则
- 引入新闻、题材和产业逻辑后的主线确认层

## 验收标准

如果后续按本设计实现，第一版应满足以下结果标准：

- 能稳定产出统一的强势股池，而不是配额式杂糅池。
- 同时覆盖趋势强票和情绪强票。
- 页面上能清楚解释一只票是因为什么强。
- 页面上能看到强势股是否集中在某些板块。
- 不要求第一版就准确归因“题材核心”，但要先把“强势识别”做对。

## 结论

本轮讨论的最终收敛方向不是继续细化旧角色体系，而是：

- 先做统一强势股池
- 再做解释性分类
- 用 `趋势通道 + 情绪通道` 保证覆盖面
- 用 `趋势强 / 情绪强 / 容量强` 保证可解释性

这是一个比现有角色优先框架更轻、更稳、也更适合第一版落地的方案。
