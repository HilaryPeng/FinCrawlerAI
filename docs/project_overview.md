# 项目说明

## 1. 这个项目是做什么的

`FinCrawlerAI` 当前已经落成一套 **A 股日频市场观察系统**。

目标不是自动交易，而是每天在收盘后自动完成：

- 抓取股票、板块、涨停、市场宽度等市场数据
- 抓取新闻与题材信息（当前主要是财联社、韭菜公社）
- 构建板块与个股特征
- 生成核心标的池
- 输出网页日报
- 同步到服务器并发送飞书提醒

这套系统更偏 **复盘、观察、筛选、跟踪**，不是高频实盘交易系统。

---

## 2. 当前主链路

项目当前主链路：

`市场采集 -> 新闻采集 -> SQLite 入库 -> 特征构建 -> 观察池 -> HTML 日报 -> 部署 -> 飞书`

展开后是：

1. 采集股票日线、板块快照、涨停池、市场宽度
2. 采集新闻、题材、异动解析
3. 统一写入本地 `SQLite`
4. 生成板块和个股特征
5. 生成每日核心标的池
6. 生成 JSON / Markdown / HTML 日报
7. 把 HTML 发布到服务器
8. 把本地 `data/` 快照同步到服务器
9. 通过飞书发送日报链接

---

## 3. 当前系统输出什么

每天的核心产出是：

- `data/db/market_daily.db`
  - 本地 SQLite 主库

- `data/processed/market_daily/market_daily_YYYYMMDD.json`
  - 结构化日报数据

- `data/processed/market_daily/market_daily_YYYYMMDD.html`
  - 页面日报

- `data/processed/market_daily/market_daily_index.html`
  - 历史日报索引页

---

## 4. 当前系统主要解决的问题

它主要解决以下几个问题：

- 每天自动汇总 A 股市场结构
- 识别当天强势板块和核心标的
- 把行情、板块、新闻、热度放到一个统一输出里
- 避免手工每天重复整理复盘素材
- 把结果自动放到网页上并发飞书

---

## 5. 当前运行模式

当前推荐模式是：

- **数据库和原始数据保留在本地电脑**
- **服务器主要负责托管网页和保存数据快照**

原因：

- 本地网络对部分数据源更稳定
- 服务器在海外时，对国内资讯站点兼容性较差
- 本地更适合作为主采集节点

---

## 6. 主要数据源

当前主要用到的来源：

- AkShare
  - 股票日线
  - 板块快照
  - 涨停池
  - 市场宽度
  - 热度/技术榜单

- BaoStock
  - 行业 membership / 行业归属

- 财联社
  - 快讯新闻

- 韭菜公社
  - 异动解析
  - 涨停原因增强

- 雪球 / THS / 东方财富部分榜单
  - 热度、关注、讨论、创新高、突破等增强特征

---

## 7. 当前自动化形态

项目目前支持两种自动运行：

### 服务器本地日跑

服务器自己跑完整链路，再发飞书。

### 本地 macOS `launchd`

本地电脑开机自启 + 工作日 21:00 自动执行：

- 本地采集和生成
- 同步数据快照到服务器
- 部署页面到服务器
- 通过服务器发飞书

---

## 8. 当前更适合做什么

现在更适合继续做：

- 稳定日跑
- 累积样本
- 完善展示
- 后续再优化逻辑和分析层

不建议现在继续无限扩字段或过早做复杂策略回测。
